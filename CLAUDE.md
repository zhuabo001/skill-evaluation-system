# 开发指导

1. 任何代码开发需求，再开始前需要向开发者确认对应的plan.md文档的位置(本项目目前都放在了项目根目录的`/v1/plans`目录下)，禁止在缺少plan.md的情况下进行开发!
2. 实际开发前，项目根目录下`/v1/progress`下新建一个名为`[plan.md名称]-progress.md`的文件用来作为任务面板，可以以4列多行的表格的形式，第一列为大任务/步骤名称，第二列为小任务/步骤名称， 第三列为任务状态，第四列为完成该任务时的commit信息和commit的哈希值；初始都为未完成，当你认为任务完成时，先比对代码修改和plan.md中的步骤描述，内容对齐后再将对应的progress.md中的任务状态标记为完成！
3. 标记完状态后进行一次commit，作为代码修改节点，方便后续回滚，将本次commit的哈希值存入到对应progress文档的第四列
4. 完成上述行为后再进行下一步开发
5. progress.md的任务状态都为'完成'后，拉起一个verification-agent专门用来比对plan.md和所有代码修改，判断是否完成了所有plan.md指示的内容，并进行代码对抗性检查
6. 检查没有问题后通知开发者进行人工验收，若发现问题务必遵循上面1-5步进行问题修复！
7. 成功后在 `/v1/records`目录下创建一个markdown文档来填入一些摘要信息，包括为了完成什么功能修改了什么文件对别的模块是否存在影响

## typescript
1. 变量必须加类型声明
2. 优先使用'unknown'而不是'any'
3. 类型不确定时优先使用泛型
4. 函数形参也需要标注类型，包括返回值类型
5. 语句结束必须使用`;`

# 重要
如果开发者在开发过程中做出动态调整，以开发者指令为准！
